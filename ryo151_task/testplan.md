
# テストプラン（テスト計画書）

## 1. テスト目的
本テストプランは、開発するCLIツールが `requirements.md` に記載された必須機能（MVP）および非機能要件を満たしていること、また `design.md` のアーキテクチャ通りにデータ保存やインターフェースが機能していることを検証することを目的とする。

## 2. テスト環境・対象
* **テスト対象:** 開発したCLIツール（実行ファイルまたはスクリプト）
* **テスト環境:** ローカルマシンのターミナル
* **データ保存先:** `~/.config/[app-name]/` ディレクトリ

---

## 3. テスト項目一覧

| # | テスト項目 | 入力・操作 | 期待する結果 | 実際の結果 | 判定 |
|---|---|---|---|---|---|
| 1 | ヘルプ表示機能の検証 | `app --help` または `app -h` を実行 | 利用可能なコマンド一覧（config, exec, status等）とオプションの説明が標準出力に表示されること | | |
| 2 | 初期設定機能（フォルダ生成） | `~/.config/[app-name]/` がない状態で `app config` を実行 | `~/.config/[app-name]/` ディレクトリが自動で新規生成されること | | |
| 3 | 初期設定機能（ファイル生成） | 続けて設定フォルダ内を確認 | フォルダ内に設定ファイル（config.json または yaml）が正しい構造で作成されていること | | |
| 4 | メイン処理の実行（正常系） | `app exec [正常なターゲット]`（または `tool-name run [input]`）を実行 | 処理が正常に終了し、期待通りの実行結果とログが標準出力に表示されること | | |
| 5 | 状態確認機能の検証 | `app status` を実行 | 現在の設定値やツールのステータスが、綺麗に整理されて画面に出力されること | | |
| 6 | 不正なコマンドの入力（異常系）| `app invalid_command` （存在しないコマンド）を実行 | アプリがクラッシュせず、適切なエラーメッセージが「標準エラー出力」に表示されること | | |
| 7 | 引数不足の入力（異常系） | `app exec` （必須のターゲットを入れずに）を実行 | 引数が足りない旨のエラーメッセージが「標準エラー出力」に表示され、安全に終了すること | | |
| 8 | 設定ファイルの不在（異常系） | 設定ファイルをわざと削除して `app status` を実行 | 「設定ファイルが見つかりません」等の分かりやすいエラーメッセージが表示されること | | |
| 9 | 設定ファイルの破損（異常系） | 設定ファイルの中身をわざと壊してツールを実行 | 「設定ファイルの読み込みに失敗しました」等のエラーメッセージが表示され、異常終了すること | | |
| 10 | 応答速度の検証（非機能要件） | `time app status` などで実行時間を計測 | どのコマンドも実行時間が **1秒以内** で処理が完結すること | | |
